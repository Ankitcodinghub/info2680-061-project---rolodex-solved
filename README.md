<strong>Description</strong>

This goal of this project is to build an information manager similar to a rolodex (see&nbsp;<a href="https://en.wikipedia.org/wiki/Rolodex">http://en.wikipedia.org/wiki/Rolodex</a>). A physical rolodex is made up of cards arranged in alphabetical order by [Last Name, First Name]. Each card contains some information, usually name, address, phone number and type of business (why the person is in the rolodex). The physical mechanism of using a rolodex is turning the group of cards and flipping through them. When the last card is read, the rolodex is then at the start of the list since a rolodex is built on a circular track.

<strong>Design Notes</strong>

For this project you‚Äôll implement a&nbsp;<strong>Card</strong>&nbsp;class (in Card.h and Card.cpp), a&nbsp;<strong>Rolodex</strong>&nbsp;class (in Rolodex.h and Rolodex.cpp), and a&nbsp;<strong>main()</strong>&nbsp;function (in main.cpp). The code in main() reads interactive commands that perform actions on its Rolodex object (like add a card,&nbsp; search for a card,&nbsp; list the rolodex cards, etc).

The&nbsp;<strong>Card</strong>&nbsp;class is used to represent a single Rolodex card. This class has std::string data members for First name, Last name, Occupation, Address (entire address can be in one string), and Phone number.&nbsp; It has get/set&nbsp; member functions to allow getting and setting of the member data values (e.g. first name), and a&nbsp;<strong>show</strong>(ostream&amp; os) function that knows how to display the card on the supplied ostream parameter.&nbsp; This class does&nbsp;<em>not</em>&nbsp;know about Rolodex functionality, it just encapsulates a single card‚Äôs information.

The&nbsp;<strong>Rolodex</strong>&nbsp;class manages a collection of Card objects. It must have a data member that is an STL container class to hold the set of rolodex Card objects (i.e.&nbsp;<em>not</em>&nbsp;a C/C++ array, but an STL container like list, vector, multiset, etc), and an associated STL iterator object to reference the ‚Äòcurrent‚Äô Card. The STL container used must be able to handle the case of duplicate names (e.g.&nbsp; two of Jim Smith, etc).&nbsp; The Rolodex member functions just manage the STL collection of Cards, and may have parameters or return values that are a Card object.&nbsp;The Rolodex class does&nbsp;<em>not</em>&nbsp;have code to read data for new cards, or printing Cards. Data input is done by code in main() and Card objects are passed into and out of the Rolodex object. Displaying cards is done by the Rolodex code calling the Card‚Äôs show() member function, passing an ostream for it to display on. Some of your Rolodex member functions might include the following :

<ul>
<li><strong>add</strong>(Card&amp; card) takes a Card object as a parameter (by ref is more efficient), adds it to the STL container member (in the appropriate spot&nbsp; so the cards are kept in alphabetical order by [last name, first name]), and sets it as the ‚Äòcurrent card‚Äô in the Rolodex by setting the&nbsp;member&nbsp;STL iterator to point at the Card just added..</li>
<li><strong>remove</strong>() removes the current card from the Rolodex‚Äôs STL container, returns it, and makes the following card the ‚Äòcurrent‚Äô card. <li><strong>getCurrentCard</strong>() returns the current Card (actually, a copy of it).</li>
<li><strong>flip</strong>()&nbsp;updates the current card position to the next Card in the Rolodex‚Äôs STL container, and&nbsp;returns that Card. If at the last card in the container, it wraps around to the first card.</li>
<li><strong>search</strong>(const std::string&amp; lastname, const std::string&amp; firstname)&nbsp; finds the requested card and sets it as the current card and returns true indicating the search found a card.&nbsp;&nbsp;If no exact matching card is found, the current card position remains unchanged and false is returned.</li>
<li><strong>show</strong>(ostream&amp; os) takes an ostream as a parameter. The Rolodex show() doesn‚Äôt do any actual output ‚Äì it just iterates through the collection and requests each card to display its contents by calling its show() member function. </ul>
The&nbsp;<strong>main</strong>() function defines and loads the Rolodex object with the starting data (by adding a series of Cards to it), and then accepts&nbsp;<em>interactive</em>&nbsp;requests that act on the Rolodex. For each interactive command, main() invokes one or more member functions on the Rolodex object. For example, you might implement the interactive commands as follows:

<ul>
<li><strong>list</strong>&nbsp;‚Äì displays the entire rolodex. It calls the Rolodex‚Äôs show(‚Ä¶) function to display the all the rolodex cards..</li>
<li><strong>view</strong>&nbsp;‚Äì&nbsp;&nbsp;displays the card at the current position in the rolodex. It&nbsp;calls the&nbsp;Rolodex‚Äôs&nbsp;getCurrentCard() function, then calls show(‚Ä¶) on the returned Card.</li>
<li><strong>flip</strong>&nbsp;‚Äì updates the current roledex position to the next card, and displays it. Flipping past the last card wraps around to the first card. It&nbsp;calls the Rolodex‚Äôs flip() function to get the next Card, then calls show(‚Ä¶) on the returned Card.</li>
<li><strong>add</strong>&nbsp;‚Äì adds a new card to the rolodex. Prompts for each field value, reads them, and creates a new Card object&nbsp;with the information, then calls Rolodex‚Äôs add(‚Ä¶) function to add the new Card to the rolodex (which puts it in the correct position in its STL container).</li>
<li><strong>remove</strong>&nbsp;‚Äì removes the card at the current position. It first&nbsp;calls the Rolodex‚Äôs getCurrentCard() function , calls show(‚Ä¶) on the returned card to display it as part of the confirmation prompt, and if ‚Äòyes‚Äô is entered to remove it, it calls Rolodex‚Äôs remove() function to remove the Card from the rolodex.</li>
<li><strong>search</strong>&nbsp;‚Äì finds and displays a card.. This command prompts for the last name&nbsp;and first name to search for, then calls the Rolodex‚Äôs search(..) function. If a matching card is found, it then calls the Rolodex‚Äôs getCurrentCard() function to get it, and then calls show() to display it.&nbsp; If no matching card is found, it displays ‚Äúcard not found‚Äù.</li>
<li><strong>quit</strong>&nbsp;‚Äì exits the program.</li>
</ul>
Note that some of the command processing code in main() requires several steps (like the add, remove, and search commands) and may invoke several member functions on the Rolodex to complete a command. This keeps the Rolodex implementation *minimal*, and the Rolodex class doesn‚Äôt do any of the prompting, input, output, etc. The Rolodex class has basic functionality to manage the collection of Rolodex cards (which is a better design).&nbsp; The Rolodex class just maintains the collection of cards (in order), and provides functionality to add, remove, search for and get cards, list the collection of cards, maintains a ‚Äòcurrent‚Äô card position, and can move the position to the next card. Specific output formatting, data entry, etc., is outside of the Rolodex class. When starting a class design, keep it as small as possible ‚Äì it‚Äôs always easier to add a new member function when there‚Äôs a proven need, vs. trying to remove functions after the class is in use..

The Rolodex class must be able to handle the cases of adding or removing a card to the beginning or end of the collection, and ‚Äòwrapping around‚Äô from the end to the beginning of the collection when moving forward from the last card to the first card.

The Standard Library std::string class should be used for the character information (no char* or char arrays).
